计算机网络第二章笔记

发表于 2019-10-09 | 分类于 计算机网络笔记 |

第二章 应用层

2.1 应用层协议原理

2.1.1 网络应用程序体系结构

  • 应用程序体系结构：由应用程序研发者设计，规定了如何在各种端系统上组织该应用程序
  • 现代网络应用程序两种主流体系结构：客户-服务器体系结构，对等（P2P）体系结构
  • 客户-服务器体系：有一个总是打开的主机称为服务器，它服务于来自许多其他称为客户的主机的请求
  • P2P体系结构：对位于数据中心的专用服务器有最小的依赖。应用程序在间断连接的主机对之间使用直接通信，这些主机对被称为对等方

2.1.2 进程通信

  1. 客户和服务器进程
    • 在一对进程之间的通信回话场景中，发起通信的进程被标识为客户，在会话开始时等待联系的进程是服务器
    • 在P2P文件共享中，当对等方A请求对等方B发送一个特定的文件时，在这个特定的通信会话中对等方A是客户，而对等方B是服务器
  2. 进程与计算机网络之间的接口
  3. 进程寻址
    • 为了标识接收进程，需要定义两种信息：主机的地址，在目的主机中指定接收进程的标识符
    • IP地址，端口号

2.1.3 可供应用程序使用的运输服务

  • 套接字是应用程序进程和运输层协议之间的接口
  1. 可靠数据传输
  2. 吞吐量
    • 带宽敏感应用
    • 弹性应用
  3. 定时
  4. 安全性

2.1.4 因特网提供的运输服务

  1. TCP服务
    • 面向连接的服务
    • 可靠的数据转送服务
  2. UDP服务
  3. 因特网运输协议所不提供的服务

2.1.5 应用层协议

  • 应用层协议定义了运行在不同端系统上的应用程序进程如何相互传递报文，特别是应用层协议定义了：
    • 交换的报文类型
    • 各种报文类型的语法
    • 字段的语义
    • 确定一个进程何时以及如何发送报文
  • Web的应用层协议是HTTP，它定义了在浏览器和Web服务器之间传输的报文的格式和序列

2.1.6 本书涉及的网络应用

2.2 Web和HTTP

2.2.1 HTTP概况

  • Web的应用层协议是超文本传输协议（HTTP），它是Web的核心
  • Web页面 是由对象组成的
  • HTTP定义了Web向客户和Web服务器请求Web页面的方式，以及服务器向客户传送Web页面的方式
  • HTTP服务器并不保存关语客户的任何信息，所以我们说HTTP是一个无状态协议

2.2.2 非持续连接和持续连接

  • 非持续连接：请求以规则的间隔周期性发出
  • 持续连接：请求间断性地一个接一个发出

2.2.3 HTTP报文格式

  1. HTTP请求报文
    • 请求行
    • 首部行
    • 空行
    • 实体体
  2. HTTP响应报文
    • 状态行
    • 首部行
    • 空行
    • 实体体
    • 状态码：
      • 200 OK：请求成功，信息在返回地响应报文中
      • 301 Moved Permanently：请求地对象已经被永久地转移了，新的URL定义在响应报文的Location：首部行中。客户软件将自动获取新的URL
      • 400 Bad Request：一个通用的差错代码，指示该请求不能被服务器理解
      • 404 Not Found：被请求的文档不在服务器上
      • 505 HTTP Version Not Supported：服务器不支持请求报文使用的HTTP协议版本

2.2.4 用户与服务器的交互：cookie

  • cookie允许站点对用户进行跟踪

2.2.5 Web缓存

  • CDN：内容分发网络

2.2.6 条件GET方法

  1. 请求报文使用GET方法
  2. 请求报文中包含一个“If-Modified-Since：”首部行

2.3 因特网中的电子邮件

  • 因特网电子邮件系统的3个主要组成部分：
    • 用户代理
    • 邮件服务器
    • 简单邮件传输协议

2.3.1 SMTP

  • SMTP是因特网电子邮件的核心
  • SMTP用于从发送方的邮件服务器发送报文到接收方的邮件服务器
  • SMTP一般不使用中间邮件服务器发送邮件，这意味着邮件不会在中间的某个服务器存留
  • 客户SMTP：运行在发送邮件服务器主机上
  • 服务器SMTP：运行在接收邮件服务器主机上
  • SMTP用的是持续连接

2.3.2 与HTTP的对比

  • HTTP从Web服务器向Web客户传送文件，SMTP从一个邮件服务器向另一个邮件服务器传送文件。持续的HTTP和SMTP都使用持续连接
  • HTTP主要是一个拉协议，用户使用HTTP从服务器拉去信息，SMTP基本上是一个推协议，发送邮件服务器把文件推向接收邮件服务器
  • SMTP要求每个报文采用7比特ASCII码格式，HTTP数据不受限制
  • 如何处理一个既包含文本又包含图形的文档：HTTP把每个对象封装在它自己的HTTP响应报文中，SMTP则把所有报文对象放在一个报文中

2.4 DNS：因特网的目录服务

  • 一个IP地址由4个字节组成，有着严格的层次结构

2.4.1 DNS提供的服务

  • 域名系统DNS：
    • 一个由分层的DNS服务器实现的分布式数据库
    • 一个使得主机能够查询分布式数据库的应用层协议

2.4.2 DNS工作机理概述

  • 单一DNS服务器的问题：
    • 单点故障
    • 通信容量
    • 远距离的集中式数据库
    • 维护困难
  1. 分布式、层次数据库
    • 有3种类型的DNS服务器：根DNS服务器，顶级域DNS服务器和权威DNS服务器

2.7 套接字编程：生成网络应用

  • 典型的网络应用都是由一对程序组成的：客户程序和服务器程序
  • 网络应用程序有两类：
    • 一类由协议标准中所定义的操作实现
    • 另一类网络应用程序是专用的网络应用程序
  • TCP是面向连接的，并且为两个端系统之间的数据流动提供可靠的字节流通道
  • UDP是无连接的，从一个端系统向另一个端系统发送独立的数据分组，不对交付提供任何保证

2.7.1 UDP套接字编程

  • 因为一台主机可能运行许多网络应用进程，每个进程具有一个或多个套接字，所以在目的主机指定特定的套接字也是必须的。当生成一个套接字时，就为它分配一个称为端口号的标识符

2.7.2 TCP套接字编程

  • TCP是一个面向连接的协议。这意味着在客户和服务器能够开始互相发送数据之前，它们先要握手和创建一个TCP连接
  • 客户生成其套接字后，发起了一个三次握手并创建与服务器的一个TCP连接。发生在运输层的三次握手，对于客户和服务器程序是完全透明的
  • 服务器在收到客户进程的连接请求后，会生成一个新套接字，专门用于特定的客户，被称为连接套接字
